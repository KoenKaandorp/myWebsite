POS Tagging:
CRF vs Softmax

Part-of-speech tagging on Italian text using a 3-layer BiLSTM — comparing a standard Softmax classifier against a Conditional Random Field decoder, with FastText embeddings pre-trained on 157 languages.

Year 2025
Type Academic
Status Completed
View on GitHub
BiLSTM · CRF · POS

Overview

Part-of-speech (POS) tagging is the task of assigning a grammatical label — noun, verb, adjective, and so on — to every word in a sentence. It is one of the foundational NLP tasks: accurate POS tags inform nearly every downstream task from parsing to information extraction.

This project trains and compares two sequence-labelling models on the Italian EAGLES corpus. Both share the same BiLSTM backbone and FastText word embeddings; the only difference is in the output layer. The first model uses a standard Softmax classifier — treating each token independently at decode time. The second replaces this with a Conditional Random Field (CRF), which models tag-to-tag transition probabilities and decodes the full sequence jointly using the Viterbi algorithm.

The Problem

A Softmax classifier assigns each token a tag independently, ignoring the fact that some tag sequences are grammatically impossible — a verb following another verb without a noun in between, for example. In practice this means the model can predict sequences that no real sentence would ever have.

A CRF explicitly models these dependencies by learning a transition matrix over tags and performing Viterbi decoding across the full sequence. The hypothesis is that this structural constraint should improve accuracy, especially on morphologically rich languages like Italian where tag boundaries are informative.

Approach

  • Data — The EAGLES corpus provides Italian text pre-split into train and test sets. Sentences are reconstructed by splitting on full-stop tokens, giving the model the sentence context it needs. Unknown words are mapped to a shared <UNK> token.
  • Embeddings — A FastText model pre-trained on Italian Common Crawl (300-dimensional) is used to initialise the embedding layer. The embeddings are not frozen — they are fine-tuned during training, allowing the model to specialise them for POS tagging.
  • BiLSTM backbone — Both models use a 3-layer bidirectional LSTM with hidden dimension 256 and dropout 0.5 throughout — between embedding and LSTM, between LSTM layers, and before the output head. Gradients are clipped to norm 5 and optimised with NAdam.
  • Softmax model — The BiLSTM output at each timestep is passed through a two-layer MLP (hidden_dim → hidden_dim/2 → tagset) with ReLU and dropout. Training uses Cross-Entropy loss with ignore_index=-100 to mask padding.
  • CRF model — The BiLSTM output is projected to emission scores, then fed into a CRF layer. Training maximises the conditional log-likelihood of the correct tag sequence; inference runs the Viterbi algorithm to find the globally optimal tag sequence given the emission and transition scores.
  • Evaluation — Both overall token accuracy and OOV (unknown word) accuracy are tracked separately, because UNK tokens are the hardest cases and reveal how well the models generalise beyond their vocabulary.

Results

Both models reach strong test accuracy on Italian POS tagging, converging quickly within the first 5 epochs. The CRF model edges ahead on both overall accuracy and — more notably — on OOV word accuracy, where the structural constraints from the transition matrix compensate for the weaker token-level signal of unknown words.

Reflections

The CRF improvement is real but modest at the overall level — about 0.2 percentage points. The more meaningful gap shows up on unknown words, where the CRF's structural knowledge of which tags can follow which provides a useful prior when the embedding signal is weak. This suggests the CRF is doing exactly what it is designed to do: regularise via tag-transition constraints when token-level evidence is ambiguous.

The UNK accuracy of around 60% is a ceiling imposed by the embedding strategy: unknown words all share the same initialisation. A natural extension would be character-level or subword embeddings (e.g. BPE or byte-level), which would give the model morphological information even for unseen words — particularly valuable for Italian, which has rich inflectional morphology. Fine-tuning a multilingual transformer like mBERT would likely push both models well above 97%.
