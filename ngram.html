N-Gram Language Model

A from-scratch n-gram language model trained on the Lord of the Rings trilogy — built up from bigrams to a full backoff model with Kneser-Ney smoothing, and deployed as an interactive text generator.

Overview

This project is a pure-Python implementation of n-gram language models, trained on J.R.R. Tolkien's Lord of the Rings trilogy. Starting from the simplest possible language model — a bigram — the project builds up through trigrams and arbitrary n-grams, then layers on two important real-world improvements: backoff and Kneser-Ney smoothing.

The end result is a text generator that produces new "Tolkien-esque" sentences entirely from statistical patterns in the source text — no neural networks involved. You can try it yourself at ngram.koenkaandorp.nl.

The Problem

N-gram models are one of the cornerstones of classical NLP — predating neural language models by decades — yet they're often only taught at the surface level: "count word pairs, compute probabilities, done." The real challenge is what happens next. A naive n-gram model breaks down immediately on unseen word sequences, which are inevitable in any reasonably-sized vocabulary.

This project set out to build the full stack from scratch: from raw text tokenisation all the way to a smoothed backoff model, without relying on any NLP libraries. The goal was to deeply understand what these models actually do before moving on to the neural successors that replaced them.

Approach

The project is structured as a progressive build, with each stage adding a meaningful capability over the last.

  • Tokenisation — A custom tokeniser handles sentence boundary tokens (<s> / </s>), punctuation, and lowercasing, turning the raw trilogy text into a clean sequence of tokens.
  • N-gram frequency tables — For a given n, the model counts how often each (n-1)-word context is followed by each word, building a nested frequency dictionary. Low-frequency entries are pruned to keep the model compact.
  • Backoff model — When a context hasn't been seen at order n, the model gracefully falls back to the (n-1)-gram model, down to a unigram baseline. This solves the zero-probability problem without discarding any information.
  • Kneser-Ney smoothing — The most sophisticated variant replaces raw continuation counts with Kneser-Ney estimates: a word is weighted not just by how often it appears, but by how many distinct contexts it completes. This gives much better probability estimates for rare words.
  • Model serialisation — Both the standard and backoff models can be saved to and loaded from JSON, making the trained model portable and usable in the deployed web app.

Results

The backoff model with Kneser-Ney smoothing produces noticeably more coherent output than the plain n-gram baseline. At higher n-values the model essentially memorises phrases from the trilogy; at lower values it becomes more creative but less grammatical. The sweet spot for fluent-sounding generation sits around n = 5–7.

The live demo at ngram.koenkaandorp.nl lets you experiment with different n values and seed phrases to see this tradeoff in action.

Reflections

Building this from scratch made the limitations of n-gram models very concrete. The core issue isn't the lack of smoothing — it's the lack of any notion of meaning. The model has no idea that "Frodo" and "hobbit" are related, or that "the ring" is the central object of the entire corpus. It just counts sequences.

That limitation is exactly what makes this a useful stepping stone. Neural language models solve it by learning distributed representations of words — embeddings — which is the natural next project after fully understanding what statistical models can and can't do.
